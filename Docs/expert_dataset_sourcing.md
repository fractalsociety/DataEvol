# Expert Dataset Sourcing Plan

This plan maps each DataEvol local expert to public bootstrap datasets plus the project-native traces that should eventually dominate training.

## Core Public Sources

Use these as bootstrap sources, then mix in DataEvol-owned traces as soon as enough examples exist.

| Source | Use | Notes |
|---|---|---|
| `lambda/hermes-agent-reasoning-traces` | agent planning, tool use, worker, manager, inspector, error recovery | Multi-turn tool execution traces. Good for agent workflow specialists. |
| `DJLougen/hermes-agent-traces-filtered` | higher-quality agent trace subset | Good for tool selection, verification, and recovery examples. |
| `PatronusAI/TRAIL` | trace debugging, error localization, early failure, regression testing | 148 annotated traces with 841 errors; small but high-signal. |
| `microsoft/orca-agentinstruct-1M-v1` | broad instruction-following, recipe generation, prompt pack generation | Large synthetic instruction set. Filter aggressively. |
| `SWE-bench/SWE-bench_Verified` | coding agent, regression tester, benchmark generator | 500 human-validated GitHub issue/PR tasks. |
| `nebius/SWE-bench-extra` | coding agent and repair workflows | 6,415 issue/PR pairs; useful for more training volume. |
| `sentence-transformers/codesearchnet` | coding, tool traces, report generation from code | Comment-code pairs across languages. |
| `allenai/scifact` | scientific method, verifier, critic | Scientific claims with evidence abstracts, labels, and rationales. |
| `ai4privacy/open-pii-masking-500k-ai4privacy` | privacy redaction, cleaner, ingestor | PII masking/redaction examples for assistant text. |
| `openbmb/UltraFeedback` | quality scorer, critic, promotion gatekeeper | Fine-grained preference and quality supervision. |
| `nvidia/HelpSteer2` | quality scoring and reward-style evaluation | Human-annotated helpfulness/correctness/coherence attributes. |
| `berkeley-nest/Nectar` | quality scorer, critic, prompt improvement | Ranked chat responses for preference learning. |
| `sentence-transformers/quora-duplicates` | deduper, duplicate detector | Labeled duplicate/non-duplicate pairs. |

## Project-Native Sources

These should be produced by DataEvol and should eventually outweigh public bootstrap data:

| Source | Generated by | Best for |
|---|---|---|
| `traces` table and raw trace files | ingest | ingestor, cleaner, classifier, router, worker, manager |
| `duplicate_clusters` and `duplicate_events` | ingest/dedupe | deduper, duplicate_detector |
| labels and scores | label/score | classifier, difficulty_scorer, quality_scorer |
| compressed traces | compress | trace_compressor, compressor |
| benchmark JSONL and manifests | benchmark builders | benchmark_generator, benchmark_task, regression_tester |
| experiment reports | evolve/experiment | mutation_evolver, regression_tester, promotion_gatekeeper |
| prompt packs and A/B results | prompts | prompt_improver, prompt_pack_generator |
| local adapter eval summaries | local_models | local_model_trainer, local_evaluator, model_mix_optimizer |
| privacy exports/redaction output | privacy | privacy_redactor, cleaner |
| integration payloads | integrations | integration_bridge, router, report_builder |

## Recommended Mix

Start each expert with:

1. 60% DataEvol-native examples once available.
2. 25% public specialist dataset examples.
3. 10% synthetic mutations generated from project failures.
4. 5% hard negative/regression examples.

For brand-new experts without native examples yet, start with 70% public + 30% synthetic, then reduce public data after the first few real DataEvol runs.

## Cautions

- Do not train directly on raw public data without converting to the exact compact JSON output contract each expert should emit.
- Keep benchmark/eval splits separate from training data, especially SWE-bench, SciFact, and TRAIL.
- Redact local traces before any export. Hugging Face warns agent traces may contain prompts, tool inputs, command output, local paths, screenshots, secrets, private code, and personal data.
- Check each dataset license before redistribution or committing derived rows.
